Introduction

What is Golf AI?

A mobile-app startup based in San Diego, CA using 3D pose-estimation technology to analyze and improve golf swings.

As 1 of 2 visual designers, I was responsible for the visual identity of the company. For the app, we received midfis from the UX team and developed hifis to handoff to the developers.

Brand Major Project 1 of 2

Challenge Point 1

How can we further develop the company’s brand?

Buzzwords

Both the UX team and users desired the app to be a positive, soothing and encouraging coaching experience with a balance of fun/credibility.

Colors

We took 4 different color-word association charts from online and did process of elimination. The winning colors with recurring characteristics were: green for growth and orange for confidence.

The specific shades were chosen with respect to a contrast-checker.

Our monochrome palette for text (primary, secondary, tertiary) referenced Adobe.

Typography

We continued to use the apps’s existing font, Lato. We adjusted line-height for text boxes to have height as a multiple of 4 (vertical rhythm based in 8pt).

We also wrote use cases for developer reference.

This was a continuously developing process as the number of screens grew.

Development Handoff

Problem point 3

According to the previous UX team, difficulties communicating resulted in inconsistent spacing.

We had pre-development meetings for devs to validate the structural logic of our hifis. For any questions about feasibility, we tried to address them beforing finishing mockups. By our third week of handoffs, we were consistently presenting 3 main aspects.

(Note: Zeplin and Figma were previously used in handoff)

Zeplin

Improving Organization

We began numbering screens (ex: Drills 01, Drills 02) and used tags for easier grouping/sorting.

Version History

I added commit messages to update if any components/screens changed.

Pseudocode

Previously, handoff for Zeplin didn’t include descriptions. I started to write notes based on CSS to demonstrate how elements should behave.

Example cases:

  • Elements positioned relative to the bottom or right
  • Using proportional height for devices (ex: this image should be 50% screen height)
  • Fixed sizes (ex: this card has a fixed height of 80pt) v. responsive sizes (ex: the banner should be 100% screen width)

Figma

Since you can see all artboards at once, we used this to make more interactive annotations and so devs could see prototypes. This is also where we placed iPhone X and 6/7/8+ appearances for reference.

Google Drive

We created folders for each screen flow and placed our assets there.
